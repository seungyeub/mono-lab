# Future Considerations & TODOs

이 문서는 프로젝트 개발 중 발생한 디자인적, 기술적 고민들과 향후 해결해야 할 주요 과제들을 기록하는 공간입니다.

## 1. 데스크톱 환경 전체 폰트 스케일업(Scale-up) 고려

**배경 (Context):**

- FAQ 섹션 리디자인 과정에서 이미지를 제거하고 1단 Full-width 구조(Route B)를 적용하는 실험을 진행함.
- 1단 구조 특성상 1440px 이상의 넓은 데스크톱 화면에서는 기존의 작은 텍스트(`text-base` 수준)가 너무 넓게 퍼져 가독성이 떨어지고 휑해 보이는 현상이 발생함.
- 이를 해결하기 위해 FAQ 아코디언의 텍스트 크기를 거대하게(`text-3xl` 등) 키워 타이포그래피 포스터처럼 연출하면 시각적 문제는 해결되나, **사이트 내 다른 섹션(Works, About 등)과의 타이포그래피 위계질서 및 일관성이 심각하게 훼손**된다는 딜레마에 부딪힘.

**고민 사항 (Dilemma):**

- 특정 섹션(FAQ)의 시각적 완성도를 위해 해당 섹션만 글씨를 뻥튀기하는 것은 근본적인 해결책이 아님. (디자인 시스템 붕괴)
- 만약 FAQ와 같은 1단 브루탈리즘/미니멀리즘 스타일을 데스크톱에서 유지하고 싶다면, **데스크톱 화면 전체(`lg` 브레이크포인트 이상)의 기본 Body 폰트 크기와 Heading 스케일을 전체적으로 일괄 상향 조정(Scale-up)하는 대대적인 작업이 필요**함.

**향후 작업 방향 (Next Steps):**

- [ ] 현재의 오리지널 폰트 사이즈를 유지하며 '2단 오프셋'이나 'Max-width 래퍼'를 사용할 것인지, 아니면 데스크톱 환경 전체를 브루탈리즘 스케일로 키울 것인지 최종 결정하기.
- [ ] 만약 전체 스케일업을 결정할 경우, `WorksSection`, `HeroSection` 등의 텍스트 크기를 모두 일괄 조정할 수 있도록 Tailwind의 글로벌 타이포그래피 토큰(예: `:root`의 폰트 스케일 변수) 또는 컴포넌트별 반응형 클래스 재정의 작업 진행하기.
